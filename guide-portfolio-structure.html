신뢰도 높은 포트폴리오 페이지 구조 예시

최초 발행: 2026-02-18 · 최종 수정: 2026-02-21

왜 페이지 분리가 중요한가

검색엔진과 AI 답변 엔진은 “누가 운영하는 사이트인지”, “문의 경로가 명확한지”, “정책이 공개되어 있는지”를 신뢰 신호로 사용합니다. 모든 내용을 홈에 몰아두기보다 역할별 페이지를 분리하면 해석이 쉬워집니다.

권장 정보 구조

  • 홈: 핵심 콘텐츠 목록과 최신 글 진입점
  • 소개: 운영 목적, 작성 원칙, 업데이트 정책
  • 문의: 협업/제보 접수 경로와 응답 방식
  • 개인정보처리방침: 수집 항목, 보관 기간, 제3자 서비스

내부 링크 설계 원칙

모든 페이지 상단 네비게이션과 하단 푸터에서 동일한 핵심 링크를 제공합니다. 이렇게 하면 크롤러가 중요한 URL을 빠르게 재발견하고 사이트 구조를 안정적으로 이해합니다.

구조화 데이터 체크

  • 페이지 유형에 맞는 Schema.org 타입 사용
  • canonical, og:url, structured data URL을 동일하게 유지
  • 문서 날짜(`dateModified`)를 실제 수정 시점과 일치
